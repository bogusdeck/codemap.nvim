*codemap.nvim*  Small competitive-programming workspace helper for Neovim

Author: bogusdeck
License: MIT

INTRODUCTION                                                    *codemap*

codemap.nvim opens a focused two-pane workspace for scratch coding and
competitive programming. It keeps shared |input.txt| and |output.txt| files in
one workspace directory and can run the current solution directly from Neovim.

INSTALLATION                                                  *codemap-install*

Example with lazy.nvim:

>
  {
    "bogusdeck/codemap.nvim",
    lazy = false,
    config = function()
      require("codemap").setup()
    end,
  }
<

CONFIGURATION                                                 *codemap-config*

>
  require("codemap").setup({
    workspace = vim.fn.stdpath("data") .. "/codemap",
    run_timeout_ms = 3000,
    default_language = "go",
    keymaps = true,
  })
<

Options:

- `workspace`: shared workspace directory.
- `run_timeout_ms`: run timeout in milliseconds.
- `default_language`: fallback filetype for unnamed buffers.
- `keymaps`: enable built-in mappings when true.

COMMANDS                                                       *codemap-cmds*

:Codemap
  Open the shared Codemap workspace.

:Run
  Run the current Codemap buffer.

:CodemapRun
  Run the current Codemap buffer.

:CodemapLanguage
  Select the language for the current unnamed Codemap buffer.

:CodemapDebug
  Debug the current Go or Python buffer with nvim-dap.

MAPPINGS                                                      *codemap-maps*

When `keymaps = true`:

- `<leader>cr`: run the current Codemap buffer.
- `<F5>`: run the current Codemap buffer.
- `<leader>cd`: debug the current Codemap buffer.
- `<F6>`: debug the current Codemap buffer.

WORKSPACE                                                  *codemap-workspace*

Codemap stores the following files under the configured workspace:

- `input.txt`
- `output.txt`
- `.codemap-build/`
